# 技术设计与任务书

## 1. 技术设计（修订：前后端目录浏览器）

### 1.1 目标
在浏览器环境中提供“可手输 + 可浏览服务器本地目录”的项目路径选择能力：
1. 后端新增目录枚举接口（仅返回目录，不返回文件）。
2. 前端 `PathPicker` 改为“输入框 + 浏览按钮 + 模态弹窗目录树导航”。
3. 最终输出绝对路径字符串，保持与现有 `projectPath` 表单契约一致。

### 1.2 后端方案
文件：`packages/server/src/routes/directories.ts`

1. 路由工厂：`createDirectoriesRouter(): Router`
2. 端点：`GET /directories?path=<absolute_path>`
3. 规则：
   - `path` 缺失或空字符串时默认 `os.homedir()`
   - 使用 `fs.readdir(targetPath, { withFileTypes: true })`
   - 仅保留目录，过滤掉隐藏目录（名称以 `.` 开头）
   - 按目录名字母排序
4. 响应数据：
   - `currentPath`
   - `parentPath`（根目录返回 `null`）
   - `entries: [{ name, path, isDirectory: true }]`
5. 错误处理：`try/catch` 返回统一 `ApiResponse` 失败包裹。

### 1.3 前端方案
文件：`packages/web/src/components/PathPicker.vue`

1. 组件对外契约保持不变：
   - Props：`modelValue: string`
   - Emits：`update:modelValue`
2. UI 结构：
   - 主体：输入框 + 浏览按钮
   - 弹窗：标题、当前路径、上级目录按钮、目录列表、底部确认栏
3. 交互流程：
   - 点击“浏览”打开弹窗并请求 `/api/directories`
   - 点击目录进入子目录
   - 点击“上级目录”进入父目录
   - 点击“确认”写回当前路径并关闭弹窗
4. 数据请求：统一通过 `httpGet`，禁止直接 `fetch/axios`。
5. 状态：加载中、空目录、加载失败均使用 i18n 文案展示。

### 1.4 i18n 设计
1. `create` 命名空间新增目录浏览相关 key（中英文双语对齐）。
2. 删除不再使用的 `create.browserNotSupported`。
