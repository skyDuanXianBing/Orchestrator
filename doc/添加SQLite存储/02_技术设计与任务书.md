# 技术设计与任务书（SQLite 持久化）

## 1. 设计目标
- 将后端运行数据持久化到 SQLite，覆盖 `tasks/phases/sse_events/blackboard_snapshots` 四类核心数据。
- 以数据库作为任务列表/详情的数据源，避免仅依赖进程内存与文件 JSON。
- 保持现有 REST/SSE 对外接口不变，内部替换为 Repository + SQLite 实现。

## 2. 模块设计

### 2.1 数据库初始化模块
- 文件：`packages/server/src/db/database.ts`
- 职责：
  - 统一创建 Node 内置 `node:sqlite` 的 `DatabaseSync` 连接（支持环境变量 `ORCHESTRATOR_DB_PATH`）。
  - 执行 Schema 初始化与索引创建。
  - 启用 `PRAGMA foreign_keys = ON`，保证级联删除生效。
  - 暴露 `initializeDatabase/getDatabase/closeDatabase` 生命周期接口。
  - 提供统一事务封装（`runInTransaction`），替代 `better-sqlite3` 的 `database.transaction()`。

### 2.2 Repository 层
- `task-repository.ts`
  - `upsertTaskState`：更新任务当前态（不覆盖已有 `pipeline_status`）。
  - `updatePipelineStatus`：状态迁移持久化（RUNNING/PAUSED/FAILED 等）。
  - `listTaskSummaries/getTaskDetail/getTaskRuntime/listTaskRuntimes`：支撑列表、详情、启动恢复。
  - `countActiveTasks/deleteTask`：健康检查与删除。
- `phase-repository.ts`
  - `upsertPhases`：将 `PhaseRuntime` 批量 UPSERT 到 `phases`。
  - `getPhaseRuntimeMap`：恢复状态机时读取阶段运行态。
- `event-repository.ts`
  - `appendEvent`：按 `task_id` 计算并写入单调递增 `seq`。
  - `listEventsByTask`：支持按 `lastSeq` 历史回放。
- `blackboard-snapshot-repository.ts`
  - `insertSnapshot`：按任务维度写入递增版本快照。
  - `getLatestSnapshot`：黑板损坏时兜底恢复。

### 2.3 黑板持久化改造
- 文件：`packages/server/src/core/blackboard.ts`
- 核心变更：
  - 从文件写入改为 SQLite 写入。
  - `write()` 每次同步更新：`tasks` 当前态、`phases` 运行态、`blackboard_snapshots` 版本快照。
  - 写入顺序固定为 `tasks -> phases -> snapshots -> tasks(version refresh)`，确保外键约束下首写不触发失败。
  - 修复 `global_context` 被覆盖为 `null` 的问题：写入前保留上一版本 `global_context`。

### 2.4 状态机改造
- 文件：`packages/server/src/core/state-machine.ts`
- 核心变更：
  - `advance/updatePhase/setCurrentPhaseIndex` 写入黑板时携带 reason。
  - 新增 `initialState` 恢复能力，支持服务重启后用 DB 状态重建运行时状态机。

### 2.5 TaskManager 改造
- 文件：`packages/server/src/services/task-manager.ts`
- 核心变更：
  - 构造函数注入三类 Repository（Task/Phase/Snapshot）。
  - 启动时从 DB 恢复任务上下文（hydrate in-memory map）。
  - `listTasks/getTaskDetail` 改为直接查询 SQLite。
  - `startPipeline/abortPipeline/review` 的状态切换同步写入 `tasks.pipeline_status`。
  - `deleteTask` 使用 DB 删除，依赖外键级联清理子表。

### 2.6 EventBus 改造
- 文件：`packages/server/src/utils/event-bus.ts`
- 核心变更：
  - 注入 `EventRepository`，每次 `emit()` 先落库再广播。
  - 提供 `listPersistedEvents(taskId, lastSeq)`，支持 SSE 重放。
  - 新增按任务订阅的 `seq + event` 通道，供 SSE 路由给实时事件补充 `id`（断线续传锚点）。

### 2.7 服务入口改造
- 文件：`packages/server/src/index.ts`
- 核心变更：
  - 启动时初始化数据库连接与所有 Repository。
  - 将 Repository 注入 `EventBus` 与 `TaskManager`。
  - 进程优雅退出时关闭 SQLite 连接。

## 3. 关键数据结构映射
- `PipelineState` ⇄ `tasks + phases + blackboard_latest_json`。
- `PhaseRuntime` ⇄ `phases`（数组字段采用 JSON 文本）。
- `SSEEvent` ⇄ `sse_events`（`data_json` 原样存储，附加 `phase_id/level/agent/message` 冗余索引列）。
- `BlackboardJson` ⇄ `tasks.blackboard_latest_json` + `blackboard_snapshots.blackboard_json`。
