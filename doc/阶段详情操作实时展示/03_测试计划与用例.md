# 阶段详情操作实时展示｜测试计划与用例

## 1. 测试目标

- 验证阶段详情弹窗可实时展示操作时间线（含顺序、去重、截断）。
- 验证 `read_files` / `changed_files` / `commands_executed` / `artifact_pointers` 汇总字段正确累积。
- 验证敏感信息脱敏与“仅元数据”落盘约束。
- 验证重启恢复后时间线与汇总字段不丢失。

## 2. 环境与前置条件

- 启动服务端与前端：`pnpm dev`
- 准备一个可触发 Read/Glob/Grep/Bash/Edit 的任务。
- 浏览器打开任务详情页，展开某阶段详情弹窗。

## 3. 手工测试用例

### 用例 1：实时时间线追加

步骤：
1. 启动任务流水线，定位到执行中的 agent 阶段。
2. 打开阶段详情弹窗，观察 operations 区。
3. 在 agent 执行期间持续观察条目变化。

期望：
- 新条目自动出现，无需手工刷新页面。
- 条目至少包含时间、类型、状态、标签。
- 每次 `BLACKBOARD_UPDATED` 后详情可刷新到最新。

### 用例 2：排序与幂等

步骤：
1. 人为制造 SSE 重连（断网后恢复或刷新 SSE 通道）。
2. 观察同一阶段时间线。

期望：
- 条目按 `seq` 升序展示。
- 相同 `op_id` 不重复显示。

### 用例 3：500 条截断

步骤：
1. 通过多轮工具操作让单阶段条目超过 500。
2. 打开黑板 JSON 与弹窗时间线。

期望：
- `operations` 仅保留最近 500 条。
- `operations_seq` 继续递增，不回退。
- 汇总字段不因截断而丢失历史去重结果。

### 用例 4：汇总字段补齐（read_files）

步骤：
1. 让 agent 执行多个 Read 操作（同路径 + 不同路径）。
2. 查看阶段详情汇总区的 `read_files`。

期望：
- `read_files` 展示去重后的路径列表。
- 与时间线中的 Read 操作目标路径一致。

### 用例 5：脱敏与安全约束

步骤：
1. 执行含敏感参数命令（如 token、password、Authorization header、URL query）。
2. 查看时间线 target/meta、黑板 JSON 持久化数据。

期望：
- 敏感值被替换为 `[REDACTED]` 或掩码。
- 不出现文件正文、grep 命中正文、stdout/stderr 全文。

### 用例 6：重启恢复

步骤：
1. 任务执行后停止服务端并重启。
2. 重新进入任务详情页查看同一阶段。

期望：
- `operations`、`operations_seq`、`read_files` 能恢复。
- 阶段详情与黑板 JSON 保持一致，不丢字段。
