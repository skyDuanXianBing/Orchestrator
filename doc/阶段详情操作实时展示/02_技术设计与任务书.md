# 阶段详情操作实时展示｜技术设计与任务书

## 1. 目标与范围

- 在阶段详情弹窗中实时展示任意子代理操作时间线，覆盖 Read / Glob / Grep / Command / Edit / Artifact。
- 每个 phase 仅保留最近 500 条操作，按 `seq` 稳定排序，基于 `op_id` 幂等去重。
- 保持现有汇总字段语义不变，并补齐 `read_files` 聚合。
- 严格执行“仅元数据落盘”，禁止保存文件正文、stdout/stderr、未脱敏敏感值。

## 2. 数据模型设计

### 2.1 Shared 类型扩展

1) `PhaseRecord` 新增（向后兼容，新增字段可选）：
- `read_files?: string[]`
- `operations?: OperationTimelineEntry[]`
- `operations_seq?: number`

2) `PhaseRuntime` 新增（向后兼容，新增字段可选）：
- `readFiles?: string[]`
- `operations?: OperationTimelineEntry[]`
- `operationsSeq?: number`

3) `OperationTimelineEntry`：
- 基础字段：`op_id`、`seq`、`ts`、`phase_id`、`agent`、`op_type`、`state`、`label`
- 可选字段：`target`、`meta`、`redaction`
- `target` 仅保存脱敏后的对象定位信息（如 path / pattern / command / url / artifact）
- `meta` 仅保存安全统计值（count、exit_code、duration_ms 等）

## 3. 服务端采集与持久化

### 3.1 采集入口

- 在 `packages/server/src/pipeline/runner.ts` 的 `forwardSdkEvent()` 中扩展 `message.part.updated`。
- 解析 `ToolPart` / `PatchPart` / `FilePart`，映射为统一 `OperationTimelineEntry`。

### 3.2 分类映射与汇总字段联动

- `ToolPart`：按工具名映射到 `read.file` / `search.glob` / `search.grep` / `exec.command` / `fs.edit` / `net.fetch` / `unknown`。
- `PatchPart`：映射 `fs.edit`，并补充 `changed_files`。
- `FilePart`：映射 `artifact.add`，并补充 `artifact_pointers`。
- 同步补齐 `read_files`：从 Read 工具目标路径去重累积。

### 3.3 幂等、排序与截断

- `op_id`：基于 phase + part 唯一键（含状态维度）生成，重复事件直接忽略。
- `seq`：由 `operations_seq` 单调递增分配。
- 时间线截断：仅保留 `operations` 最新 500 条，`operations_seq` 不回退。

### 3.4 脱敏与禁止项

- 命令脱敏：secret flags、环境变量赋值、Authorization header、URL 敏感 query 参数、长 token 串掩码。
- 严禁写入：文件内容、grep 命中文本、stdout/stderr、敏感 header 原值。
- 所有时间线 `target/meta` 在入库前统一走脱敏与裁剪。

### 3.5 持久化与重启恢复

- 不改现有 `phases` 表结构，采用最小变更方案。
- 通过 `blackboard_latest_json` 保存完整时间线与 `read_files`。
- 任务恢复时优先从 blackboard JSON 还原 phase runtime，避免新字段在重启后丢失。

## 4. 前端展示设计

### 4.1 PhaseDetailModal

- 汇总区新增 `read_files` 列表。
- 新增“操作时间线”区：
  - 每项显示 `ts`、`op_type`、`label`、`state`。
  - 支持展开查看 `target/meta`（仅脱敏元数据）。
  - 按 `seq` 排序展示，支持最多 500 条滚动查看。

### 4.2 实时刷新机制

- 继续复用 `BLACKBOARD_UPDATED` 触发 `TaskDetail` 刷新详情。
- SSE 为主、轮询降级，不增加并行重复订阅。

### 4.3 i18n

- 组件新增可见文案全部接入 `vue-i18n`。
- 同步补充 `zh-CN.ts` 与 `en.ts` 文案键。
