# 阶段详情实时展示｜技术设计与任务书

## 1. 目标与约束

- 目标：让 Phase 详情弹窗在流水线运行中实时显示 `status`、耗时、`changed_files`、`commands_executed`、`artifact_pointers`，并在阶段结束后稳定冻结。
- 实时口径：SSE 为主，Polling 为降级；SSE 恢复后立即停止轮询。
- 约束：最小改动，不改测试；前端可见文案继续走 i18n；后端保持现有 ESM 与状态机写盘路径。

## 2. 现状问题

- `TaskDetail.vue` 未监听 `PHASE_STARTED`，阶段开始时 UI 刷新不及时。
- 轮询当前常驻（5s），不是 SSE 断线降级模式。
- `PhaseDetailModal.vue` 虽计算 `Date.now()`，但缺少响应式 tick，运行中耗时不会每秒刷新。
- `useSSE.ts` 在 `on()` 与 `connect()` 双路径加监听，存在重复回调风险。
- `runner.ts` 对 SDK `file.edited` 仅发事件，不增量写入 `changedFiles`；`command.executed` 未处理。

## 3. 方案设计

### 3.1 Web 侧

1) `TaskDetail.vue`
- 刷新事件补齐：将 `SSEEventType.PHASE_STARTED`（并补 `PIPELINE_STARTED`）加入触发 `fetchTaskDetail()` 的事件列表。
- 轮询降级化：
  - 轮询间隔改为 2000ms；
  - 监听 `sse.connected`：`true` 时 `polling.stop()`，`false` 时 `polling.start()`。

2) `PhaseDetailModal.vue`
- 增加 1 秒响应式计时器：当 `status===IN_PROGRESS && started_at && !finished_at` 时每秒更新一次 tick。
- `finished_at` 写入后停止计时器，耗时固定为 `finished_at - started_at`。
- `started_at` 缺失或非法时显示 `notAvailable`。

3) `useSSE.ts`
- 将事件处理器存储从 `Array` 改为 `Set`，同一 handler 去重。
- 移除 `on()` 内对 `eventSource.addEventListener` 的即时绑定，统一走 `connect()` 注册的单一路径分发，确保每个事件每个 handler 只触发一次。

### 3.2 Server 侧

1) `runner.ts / forwardSdkEvent()`
- `file.edited`：
  - 读取当前 phase 的 `changedFiles`；
  - 对文件路径去重后追加；
  - `stateMachine.updatePhase()` 写入；
  - 发送 `BLACKBOARD_UPDATED`。
- `command.executed`：
  - 从 `properties` 组装命令字符串（`name + arguments`）；
  - 去重后追加到 `commandsExecuted`；
  - `stateMachine.updatePhase()` 写入；
  - 发送 `BLACKBOARD_UPDATED`。

## 5. 风险与回退

- 高频 SDK 事件会增加 `updatePhase` 写盘次数；本次保持最小实现，后续如有性能瓶颈再做批量合并。
- 若实时事件异常，仍可依赖 2s 轮询保证最终一致性。
